## Camera Setup
:::danger
Raspberry Pi 5 的攝像頭連接是使用 15-pin 的 CSI 接口，一般模組是 22-pin，需要一條 15pin-22pin 的轉接線。
:::
設置部分可參考 
* <a href="https://www.raspberrypi.com/documentation/accessories/camera.html#install-a-raspberry-pi-camera" target="_blank" rel="noopener noreferrer">Raspberry Pi Camera Documentation</a>
* <a href="https://www.raspberrypi.com/documentation/computers/camera_software.html" target="_blank" rel="noopener noreferrer">Raspberry Pi Camera Software Documentation</a>

>[!Note] 相機型號
>我用的相機型號是 Raspberry Pi Camera Module 3 Standard (75°)，具體參數設定請參考自己的型號。

要在 Raspberry Pi 5 用攝像頭，可以用官方自帶的 rpicam-apps、gstreamer、libcamera 或者最底層的 v4l2。
**難易度**：rpicams-app < gstreamer = libcamera < v4l2

>[!note] Note
如果追求非常流暢的串流體驗或者想接觸底層，可以用 v4l2。

用 rpicam 測試攝像頭：
```bash
rpicam-still -t 0
```

autofocus（如支援）：

```bash
rpicam-still -t 0 --autofocus-mode=continuous --flush --framerate 60
```

### 安裝 gstreamer
```bash
sudo apt-get update
sudo apt-get install gstreamer1.0-tools libgstreamer1.0-dev libgstreamer-plugins-base1.0-dev gstreamer1.0-plugins-base-apps -y
```

### 安裝 libcamera
```bash
sudo apt install libyaml-dev python3-yaml python3-ply python3-jinja2 libpython3-dev pybind11-dev python3-libcamera
git clone https://git.libcamera.org/libcamera/libcamera.git
cd libcamera
meson setup build --buildtype=release -Dgstreamer=enabled -Dpycamera=enabled
ninja -C build install
```

新增 `PYTHONPATH` 和將 `GST_PLUGIN_PATH` 放入 `~/.bashrc`：
```bash
export PYTHONPATH=$PYTHONPATH:/usr/lib/python3/dist-packages
export GST_PLUGIN_PATH=$HOME/libcamera/build/src/gstreamer:$GST_PLUGIN_PATH
```

執行：

```bash
source ~/.bashrc
```

看看 `libcamerasrc` 有哪些可用的 properties：

```bash
gst-inspect-1.0 libcamerasrc
```

用 gstreamer 測試：
```bash
gst-launch-1.0 libcamerasrc ! video/x-raw,format=NV12,width=1280,height=720,framerate=60/1 ! queue max-size-buffers=1 leaky=downstream ! glimagesink sync=false
```

autofocus:

```bash
gst-launch-1.0 libcamerasrc af-mode=continuous ! video/x-raw,format=NV12,width=1280,height=720,framerate=60/1 ! queue max-size-buffers=1 leaky=downstream ! glimagesink sync=false
```

## OpenCV Setup
同上，可從 <a href="https://hackmd.io/@EthanChan/jetson-nano-setup-guide#%E5%88%AA%E9%99%A4%E8%88%8A%E7%89%88-OpenCV" target="_blank" rel="noopener noreferrer">Jetson Nano Setup Guide</a> 這裡開始做，由於樹莓派沒有 CUDA，CUDA 的步驟都可以跳過。

```bash
sudo apt install python3-libcamera
```
CMake 的地方可以把 CUDA 相關的都移除，以下設置建議開起來：
```
-D WITH_OPENCL=ON \
-D ENABLE_NEON=ON \
-D WITH_OPENGL=ON \
-D WITH_FFMPEG=ON \
-D WITH_GSTREAMER=ON \
-D WITH_TBB=ON \
-D BUILD_TBB=ON \
-D WITH_V4L=ON \
-D WITH_LIBV4L=ON \
```

### 增加 Virtual (swap) memory

檢查系統版本：

```bash
lsb_release -a  # Raspi 5 已經升級成 Trixia
free -h  # 初始為 2GB
swapon --show   # 查看 swapping 底層核心，通常是 zram
```

這裡示範的是 `rpi-swap + zram0`

```
sudo mkdir -p /etc/rpi/swap.conf.d

sudo tee /etc/rpi/swap.conf.d/99-zram-size.conf > /dev/null <<'EOF'
[Main]
Mechanism=zram

[Zram]
FixedSizeMiB=4096   # 這一行決定多少 GB
EOF
```

重啟：

```bash
sudo reboot
```

檢查最新的 swap size：

```bash
free -h
swapon --show
zramctl
```

### OpenCV Read Camera Input
OpenCV 在 Linux 上沒辦法直接用 `/dev/video0` 讀取 CSI camera，主要有三種方法：**GStreamer**、**V4L2**、**picamera2**

V4L2 的方法可參考 <a href="https://blog.arducam.com/faq/opencv-v4l2-python-rpi/" target="_blank" rel="noopener noreferrer">https://blog.arducam.com/faq/opencv-v4l2-python-rpi/</a>

這裡使用 GStreamer 的方法：
```python
import cv2

pipeline = (
    "libcamerasrc "
    "af-mode=continuous"
    "camera_name=/base/axi/pcie@1000120000/rp1/i2c@88000/imx708@1a ! "
    "video/x-raw,format=NV12,width=1280,height=720,framerate=30/1 ! "
    "queue max-size-buffers=1 leaky=downstream ! "
    "videoscale ! "
    "videoconvert ! "
    "video/x-raw,format=BGR,width=1280,height=720 ! "
    "appsink caps=video/x-raw,format=BGR drop=true max-buffers=1 sync=false"
)

cap = cv2.VideoCapture(pipeline, cv2.CAP_GSTREAMER)

while True:
    ret, frame = cap.read()
    if not ret:
        break

    cv2.imshow("frame", frame)
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
```
這裡要注意的是要指定 pipeline 的 `camera_name`，可使用 `rpicam-hello` 或 `rpicam-still --list-cameras` 來獲得：
```bash
rpicam-hello
```
找到一行 `Registered camera ...` 或者 `Adding camera '...'`，`...` 的部分就是 camera_name

#### picamera2
picamera2 是樹莓派官方提供的 Python OpenCV library 
```bash
pip install picamera2
```
```python
import cv2
from picamera2 import Picamera2

# Initialize and configure the camera
picam2 = Picamera2()
config = picam2.create_preview_configuration(main={"format": 'XRGB8888', "size": (640, 480)})
picam2.configure(config)
picam2.start()

while True:
    # Capture frame as a NumPy array (directly compatible with OpenCV)
    frame = picam2.capture_array()

    # Display the frame
    cv2.imshow("Raspberry Pi 5 Camera", frame)

    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

picam2.stop()
cv2.destroyAllWindows()
```

## Pi 5 優化

修改 Pi 5 的一些設定可以優化系統效能，可以更好的應對 OpenCV 這種需要效能的程式。

### SDRAM

```bash
sudo rpi-update  # 更新 Pi's firmware (y to confirm)，rpi-update 會安裝實驗性核心，操作前請務必備份重要資料
sudo rpi-eeprom-config -e

# 加入下面這一行：
SDRAM_BANKLOW=1
```

重啟：

```bash
sudo reboot
```

### NUMA Emulation (Non-Uniform Memory Access)

```bash
sudo apt full-upgrade -y
sudo apt install numactl
```

查看 NUMA 狀態：

```bash
dmesg | grep -i NUMA
```

>[!note] Note
新版 Pi OS 會自動把 `numa=fake=8` 放到 kernel command line \
重點查看這一行：mempolicy: NUMA default policy overridden to 'interleave\:0-7\

查看目前 NUMA 的 policy settings 和 resource bindings：

```bash
numactl --show  # 最新的 Pi OS 已經默認 policy: interleave
```

查看 NUMA 的 hit 和 miss：

```bash
numastat
```

### 超頻

這一步有可能為會樹莓派帶來風險，但在合理範圍內超頻還是沒問題的，有人曾經測試超頻到 3.4GHz，但如果你不知道自己在幹嘛，**don't do it**.

#### 風扇

不管有沒有超頻，都建議加裝 active cooler。

超頻前先把風扇速度拉到 100%：

```bash
pinctrl FAN_PWM op dl  # op 表示 output，dl 表示 drive low
```

如果要關閉風扇就是：

```bash
pinctrl FAN_PWM op dh
```

自動模式：

```bash
pinctrl FAN_PWM a0
```

>[!Tip]
dh 和 dl 的特性和 PWM 有關，有興趣的再去了解。

超頻前，確保電源供應在 5V3A 以上，因為超頻需要更多的電。

打開設定檔修改：

```bash
sudo nano /boot/firmware/config.txt

# 把下面三行加在最後
arm_freq=2600
gpu_freq=900
over_voltage=2

# 如果要永久修改風扇參數，才加入下面三行。pinctrl FAN_PWM 不會在開機時生效。
dtparam=fan_temp0=40000   # 40°C 開始進入第一段風扇
dtparam=fan_temp0_hyst=5000  # 降到 35°C 以下才退出第一段，避免風扇一直啟停
dtparam=fan_temp0_speed=180  # PWM 速度，範圍 0–255

# 重啟
sudo reboot
```

### 效能模式

```bash
echo performance | sudo tee /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor
```
