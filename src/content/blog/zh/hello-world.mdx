## 關於這篇

2009 年到現在，17 年，也代表立志讀 IT 的夢在往 18 年的路上。那個從 kampung 走向大城市的小孩，沒有 Mark Zuckerberg  式的啟蒙。從第一次寫 code 到現在，也已經快 6 年，不長不短。正如標題，第一個程式是用 python 寫的 hello world。對於到底要先學哪個語言，其實沒有一定的答案，也許讀完這篇，你就有了答案。

## 高中

回想高一，對程式沒有概念，就直接用 PyQt5 寫了第一版的辯論計時器，很陽春，但很有成就感。在那個 AI 還沒有這麼強的時代，google 和 stackoverflow 是唯一真理。後來回看以前寫的，上萬行的 code，雖然結構很爛（畢竟基本功都沒打好就想出家），但不得不讚嘆自己怎麼有辦法寫出來。

高二開始寫網站，從最初的 HTML/CSS，到後來的 JavaScript，這段過程直接開啟了我跨越 Desktop 和 Web 的旅程。當時的我對於前端和後端的概念還很模糊，不知道什麼是靜態，什麼是動態網站。

高三上課學了 MIT App Inventor，想到小學用的 Scratch，雖然不知道為什麼高三要學這個，但對沒有基礎的人來說，這何嘗不是一個好方法。跟 App Inventor 打了一年交道，最後用它做了藍牙連線版的中國象棋，過程很漫長，但也很有趣，成就感爆棚。

突然想到高三還有課室佈置，特地為這個活動用 AppSheet 做了一個 app，反正就是可以用來點名、記錄器材、工具、人員分工的 app。一開始也沒有這個打算，就只是因為敏姐的原則性和對細節的要求，就嘗試做一做。不過呢，幸好當時有用 AppSheet，才有後來自己專屬的記帳 app 誕生。

哦，還有，因為良哥的數學作業都要放在 pdf 交上去，所以為了方便，我直接用 PySide2 寫了 converter，有一些基本的轉換功能，印象中有放出去給大家用？

高中三年，發生的事情太多了，也是這三年的摸索，慢慢成就了半桶水，什麼都懂一點，但又什麼都不精通的小孩。

## Gap Year
2023 年，高中畢業後將近 9 個月的 gap，對 coding 世界的認識依然很小。這段時間，大家的目標都一樣 — 申請大學。有一句話叫「選你所愛，愛你所選」，我想，不管什麼時候，我都想把這句話送給自己。選擇了就走下去，人生沒有一條路是白走的。

9 個月的空檔，家裡也發生了很多事，有些人讀大學前都在快樂時光，有些人提前進行上大學前的能力補強。我既不是前者，也不是後者，剛剛好在中間。每個人都有自己的一套生活模式，不一定要跟別人比較，
<p style={{textAlign: 'center'}}>「You know who you are, what you do, how you live」</p> 這比跟別人比較還重要。

## 大學

普遍上，資工系最先開始教的程式語言就是 C / C++，對於沒有程式經驗的人，這段學習的路是痛苦的，畢竟有太多底層邏輯的技術名詞（pointer、types...），但對於學過  Python 的人，就算寫過，但要是不知道底層內容是什麼，也要重新理解。相近的詞，如 `import`、`include`、`enum` 等等就比較好理解。但事實是，Python 的底層是 C，所以很多概念當然相近，而且現代程式語言的語法就算不同，但做的東西都是一樣的。

C / C++ 讓你了解底層邏輯架構，Python 讓你用低學習成本學 coding，所以其實沒有對錯。

如果真的要深入理解程式設計的底層原理，就去寫組合語言，演算法和資料結構都手刻，這樣才會對架構優化有新的認知。也許可以用 AI 寫，但不如自己寫，可能 AI 寫的是最優的，自己寫的東西不見得是，但至少你知道為什麼這樣寫。

在大學，遇到了很多貴人，得到了一些機會，參加了一些比賽和專案，至於做了什麼就不多說了，首頁的專案欄都有。

## 對 IT 趨勢的看法

目前最熱門的，AI 當紅，但這代表只需要學 AI 嗎？這麼想的話，那還是往別的路走吧。除了 AI，其它熱門的還有資安和物聯網，資安跟 AI 一樣是全球熱門，都很需要能力、興趣和持續的研究。軟體工程 / 軟件工程 / Software Engineering，這三個都指同一件事，只是用詞不同。SE 不只是寫 software，其實也需要知道 hardware。放到今天來看，沒有哪一個領域是可以獨自存在而不互相影響的，舉個例子，AI 很強，但沒有資安，所有資料都是裸奔；AI 很強，但需要工程性、實務性的方法去落地，這套工作流就是 SE 在做的。物聯網呢？看看特斯拉就知道了。

不管哪一個，**可以做研究，也可以做應用**。

## How about AI?
現在的 AI 比起那個什麼都要手刻，或者去 google 找資料的時代，已經天差地遠。AI 可以做到很多事（廢話），但寫 code 這件事，不是下一句 prompt 就可以拍拍屁股全盤接收，也不是寫了一個看起來 LGTM (Looks Good To Me) 的 code 就可以發 pr。如果知其然而不知其所以然，那 debug 的時間會比寫 code 還久，甚至可能 debug 不出來。交給 AI debug？當然可以，但你永遠不知道你的架構有多屎。

學新技術的時候，AI 絕對是好幫手，AI 的時代，不要依賴，不要害怕，**要學會駕馭**。AI 是工具，不是魔法，**不會用工具的人，註定被淘汰**。保持好奇心，保持獨立思考，學會批判精神。

<p style={{textAlign: 'center'}}>「人類精神產物的價值永遠比 AI 的更高」</p>

## 結語

從一個什麼都不懂，不懂前後端、API、DB、framework、OOP、MVC、HTTP request...的菜雞，到今天有能力開發小系統，再回頭看，好像真的比當初的自己還厲害了。這條路還有很多東西要學，要做，要體驗。不需要為了追趕潮流或趨勢而放棄自己的興趣和熱情，永遠不要覺得自己很厲害，**選你所愛，愛你所選**。
