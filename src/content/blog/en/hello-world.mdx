## Why define boundaries first

When scope expands, clear module boundaries reduce coupling and keep the codebase easier to evolve.

```ts
type Requirement = {
  id: string
  domain: 'routing' | 'data' | 'ui'
  owner: string
}
```

## A practical sequence

1. Define the domain model and core fields
2. Design route responsibilities
3. Componentize the UI and connect delivery flow
